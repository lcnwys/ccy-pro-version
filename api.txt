开放 API 接口文档
一、鉴权（Authentication）
1.1 概述
调用开放 API 时，请使用平台为您分配的 API Key 进行身份校验。所有需要鉴权的接口均通过标准 HTTP 头传递密
钥；
1.2 获取 API Key
API Key 由平台 用户中心 => API 接口 => 新增API接口 操作申请
1.3 请求头格式
在每次 HTTP 请求头中的取值形如： Authorization: Bearer <YOUR_API_KEY> （将 <YOUR_API_KEY> 替换为实际
密钥）。
请求头一览
1.4 Base URL（服务根地址）
固定域名为： https://api.chcyai.com 。
请将下文各接口路径拼接在该根地址之后（例如： https://api.chcyai.com/v1/files/uploads ）。
1.5 鉴权失败时的表现
当 API Key 缺失、格式不符合要求或已失效时，接口将返回错误响应。请根据 HTTP 状态码与响应体中的 message
排查问题；常见 code 为 PARAM_ERROR 。
情况 建议处理
密钥无效或已停用 在平台核对 Key 是否正确、是否仍启用，必要时重新生成

1.6 安全建议
仅在服务端或受控环境保存 API Key，通过 HTTPS 调用本服务。
定期轮换密钥；泄露后立即在平台作废并更换。
勿在日志、错误上报、截图中输出完整 Key。
